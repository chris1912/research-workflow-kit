用途、受众与公开包差异 / Purpose and scope

做什么

可追溯编排：问题 → 文献与证据 → 方法/写作/编辑 → 结论型可读交付。产出模板化、可审计文件。

面向谁

需要可重复研究路线的科研人员、开题/申报写作者，以及在智能体环境中加载本仓库技能的用户。

与完整实验室

本包是轻量一级层；完整实验室可含更重引擎与外部方法套件。本手册只用仓库相对路径与已暴露能力。

Markdown / 结构化产物是权威源；HTML 是导航与可读交付层。可选后端全部非必需。

默认主路线 / Default route

01 问题与范围拆解指南、研究问题、纳入排除条件。
02 检索与发现建立主路由；第二提供方仅作可选盲路由。
03 合并核心归一化、去重、复筛，形成 MERGED_CORE_PAPERS.md。
04 优先全文选择 3–8 篇关键文献，合法 OA 优先并记录获取结果。
05 证据综合解析、证据卡、反证、文献图谱和综合判断。
06 写作与编辑长文起草、学术编辑、术语/文风检查。
07 HTML 交付结论型运行提供入口页、关键页面和证据边界。
08 人工确认Markdown/结构化文件保持权威，提交前人工复核。

可选后端、第二路由或外部模型缺失时，主路线仍应完成，并在 literature_route_status.json 中记录回退原因。

一级技能总览 / First-party skills

  • skills/workflow-router — 阶段不明时的全局路由
  • skills/proposal-research — 开题/申报深研、双路、合并核心、优先全文、HTML 合约
  • skills/proposal-iteration — 反馈诊断与小步修订
  • skills/research-methods — Essential Core 方法/完整性/评审入口
  • skills/longform-writing — 长文结构与分节起草
  • skills/academic-editing — 保守学术编辑与术语统一
  • skills/prose-lint — 确定性文风检查（非证据核验）

每项技能的输入、输出、示例请求与交付物路径见 详细教学手册（技能章节与可复制示例专节）。

证据边界 / Evidence boundary

claim extract inference uncertainty missing blocked human-confirmed

本仓库提供可追溯编排，不提供现成科学结论。不得编造论文、DOI、数据、实验结果、统计量或引用支持关系；缺全文或无法核验时必须标注限制，不能用摘要冒充全文方法或性能证据。

方法核心 / Essential Core

skills/research-methods 是本仓库第一方 essential_core 入口，包含合约、模式注册表、模板和可选离线 runtime。研究问题、深研、系统评价、论文方法、引用完整性和同行评审目前为 parity: partial；E4 管线/实验/optional runtime 为 parity: not_started。

部分对等不等于完整 Academic Research Skills 行为对等，也不代表真实多进程编排。完整 ARS 套件是可选外部材料，不随本仓库捆绑，也不是运行必需依赖。

申报书深研使用 proposal-research；方法协议使用 research-methods；写作栈使用 longform-writing → academic-editing → prose-lint。

交付门 / Delivery gates

  • 收敛后下游唯一权威是 MERGED_CORE_PAPERS.md；core_papers.md 仅作合并前或单路线兼容模板。
  • 优先全文只做合法 OA-first 尝试，记录 fulltext_acquisition.csv；无法获取时建立 FULLTEXT_MANUAL_ACTION_REQUIRED.md，不描述付费墙绕过。
  • 结论型运行需 UTF-8 HTML 入口、3–8 个决策关键页面、响应式/打印导航、可见证据边界和本地链接检查；纯探索可不触发此门。
  • 发布前不得包含密钥、个人研究材料、虚拟环境、缓存、二进制或无关生成物。
