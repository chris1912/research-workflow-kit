## Upgraded public gates

- Prefer repository-relative links only in tracked files.
- Treat secondary literature discovery as optional. Finish the primary route when
  secondary provider, CLI, auth, discovery, model, quota, network, or task is
  unavailable, and record the fallback in `literature_route_status.json`.
- After convergence, use `MERGED_CORE_PAPERS.md` as the sole downstream authority.
  Keep `core_papers.md` only as a pre-merge or single-route compatibility file
  that states the authority transition.
- For priority full text, select 3–8 papers, attempt lawful OA-first acquisition,
  log attempts, deep-read readable text, and open a manual queue on failure.
  Never describe paywall bypass.
- For conclusion-bearing runs, complete the HTML delivery checklist: UTF-8 entry
  page, 3–8 decision-critical pages, evidence-boundary text, responsive/print
  navigation, and local-link checks. Markdown/structured artifacts remain
  authoritative.
- Route systematic-review, integrity, and peer-review protocol tasks through
  `skills/research-methods`. Packaging mode is first-party `essential_core`
  (contracts, mode registry, opt-in runtime, dual agent surfaces). E2 RQ/deep/SR
  method depth is operational (`parity: partial`); E3–E4 protocol bodies remain
  `parity: not_started` and must not be advertised as complete ARS parity. The
  full Academic Research Skills suite is optional external material and is not
  bundled or required here.
- Keep the kit useful when every optional adapter is unset.
