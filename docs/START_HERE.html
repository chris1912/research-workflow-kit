本页是 Research Workflow Kit 的详细教学手册。首次进入请先看 项目轻量入口。此处解释技能何时使用、默认路线、产物路径、证据规则、安装验证，以及可复制示例请求。

最省事的起始请求：
“请使用 Research Workflow Kit（workflow-router）。我的目标是：[目标]。已有材料：[说明]。请先判断阶段、选择 skill、说明证据边界与必做/可选步骤，再执行。不要编造文献、DOI、数据或实验结果；结论性交付请完成 HTML 清单。”
OVERVIEW

1. 用途、受众与公开包边界

Research Workflow Kit 是一套可追溯的研究编排层：把研究问题路由到文献发现、证据处理、综合、长文起草、学术编辑与文风检查。它帮助新研究者知道该用哪个 skill、期望哪些文件、何时必须停下来做人工确认。本项目入口是 $scientific-workflow-lab-lite，与完整版 $scientific-workflow-lab 分开。

做什么

  • 阶段判断与技能路由
  • 指南/问题拆解与检索策略
  • 主路线文献发现与可选盲法次路线
  • 合并核心、优先全文、证据卡与反证
  • 长文、编辑、文风与结论型 HTML

不做什么

  • 不提供现成科学真相或“已验证结果”
  • 不保证引文正确或替代专家审阅
  • 不捆绑大型搜索/解析/模型引擎
  • 不描述付费墙绕过
  • 不宣称完整 ARS 行为对等

受众

  • 研究生与青年研究者
  • 课题/申请书准备者
  • 需要可审计代理工作流的写作者
  • 审查公开合约的维护者

公开包 vs 完整实验室

  • 公开包：一级适配器、技能合约、模板、可选离线 methods runtime
  • 完整实验室：可含更重本地工具链与私有手册（不在本仓库）
  • 本手册只写本仓库真实能力与路径
  • 可选后端全缺时仍须完成主路线并显式回退
权威层级：Markdown / JSON / CSV 等结构化产物是权威源；HTML 是人类可读导航与结论交付层；Word/PDF（若生成）是后期包装，默认不进入公开树。

2. 先路由，再执行

Skill 决定问题怎么拆、先做什么、何时可进入下一阶段；适配器/可选后端负责发现、解析、证据问答、图谱、综合或文风检查。后端缺失时返回显式 missing/blocked，不静默换用不相关工具。

PIPELINE

3. 默认主路线：从问题到交付

01 指南与问题guide_decomposition、范围、候选方向评分
02 检索策略search_strategy.md；关键词、查询族、收敛标准
03 双路由预检literature_route_status.json；secondary 可选
04 文献发现主路线必做；可选盲法次路线候选池
05 合并核心归一化/去重/复筛 → MERGED_CORE_PAPERS.md
06 优先全文3–8 篇 OA-first、获取日志、深读、人工队列
07 证据与综合证据卡、领域图、claim 核验、反证、综合
08 起草与润色longform-writing → academic-editing → prose-lint
09 HTML 交付门结论型运行：入口 + 3–8 决策页
10 打包与人工确认MD 权威；提交前 human confirmation

必做与可选 / Required vs optional

阶段中文状态
Guide / question parse指南与问题拆解深研/开题类运行：必做
Primary literature discovery主路由文献发现声称证据时：必做
Merged or primary-final core合并或主路由终稿核心表收敛后：必做权威
Evidence cards & uncertainty证据卡与不确定性强论断起草前：必做
Human confirmation人工确认门提交级结论前：必做
Secondary discovery第二路由发现可选加速；缺则 primary_only
External adapters / engines外部适配器与引擎可选；未配置须显式 missing
Priority full-text acquisition优先全文获取全文在范围内：必做尝试集；仅 OA-first
HTML conclusion package结论型 HTML结论性运行：必做；纯探索：可选
回退原则：第二路由、CLI、鉴权、发现、模型、配额、网络或任务不可用时，主路线必须完成；在 literature_route_status.json 写入 primary_only 或 primary_only_after_secondary_failure 与明确原因。
SKILLS

4. 一级技能：何时用、输入、输出、示例

本仓库提供 1 个项目级总路由和 7 个专业技能。示例请求使用中文，可直接复制到代理会话；方括号替换为实际材料。

4.0 scientific-workflow-lab-lite — 本项目轻量级总路由

何时使用：用户不知道选哪个 skill、任务跨多个阶段，或希望先看到完整研究路线、证据边界和交付门禁时。

期望输入研究目标、已有材料、联网/后端限制、输出格式和人工确认要求
核心动作读取项目入口与本手册，判断阶段，再把任务交给 workflow-router、proposal-research、research-methods、proposal-iteration、longform-writing、academic-editing 或 prose-lint
关键输出选定路线、必做/可选步骤、证据边界、预期产物、风险和下一个人工决策点

中文示例：「请使用 $scientific-workflow-lab-lite。我的目标是：[目标]，已有材料：[材料]。先判断阶段、选择专业 skill、说明证据边界与交付门禁，不要直接编造结论。」

项目契约：skills/scientific-workflow-lab-lite/SKILL.md · 宿主元数据：agents/openai.yaml

4.1 workflow-router — 全局路由

何时使用：任务跨多阶段、不确定 skill、需要先声明证据边界与必做/可选步骤时。

期望输入目标、已有材料、时限、是否允许联网检索、输出偏好
核心动作读详细手册、判断阶段、选择 skill/适配器、声明边界与门禁
关键输出路线说明、阶段清单、风险、下一步、手册链接

中文示例：「请使用 workflow-router。我的目标是为课题写可追溯的文献与方案骨架；已有一份研究设想草稿。请先判断阶段并给出 skill 路线，不要直接成稿。」

契约：skills/workflow-router/SKILL.md

4.2 proposal-research — 开题/指南优先深研

何时使用：从指南、开题材料、宽方向出发，建立检索、核心文献、证据卡与参考方案时。

期望输入指南/问题、背景、关键词、约束、已有数据说明（可无）
核心动作拆解→策略→主/可选次路线→合并核心→OA 全文→证据→起草门
关键输出见第 5 节产物清单；模板在 skills/proposal-research/templates/

中文示例：「请使用 proposal-research。指南主题：[粘贴]。请拆解约束并生成 3 个候选方向评分，再写 search_strategy.md；次路线若不可用请 primary_only 继续。不要编造文献。」

契约：skills/proposal-research/SKILL.md

4.3 proposal-iteration — 草稿迭代与修订循环

何时使用：已有申请书/方案草稿，需要把“不够好”翻译成可选修改策略并小步修订时。

期望输入现有草稿路径/正文、反馈意见、不可改动的硬约束
核心动作冻结版本→分轴诊断→小修订→复检→编辑与 lint
关键输出诊断矩阵、修订日志、新版本草稿、未决风险、检查结果

中文示例：「请使用 proposal-iteration。我觉得立项依据‘不够科学’，但说不清。请先给出 3–5 个互斥修改选项，先不要大改正文。」

契约：skills/proposal-iteration/SKILL.md

4.4 research-methods — 方法 / 完整性 / 评审 Essential Core

何时使用：研究问题收敛、深研协议、系统评价规划、引用完整性、手稿模拟审稿、论文模式规划时。申报书双路线/全文/HTML 仍归 proposal-research。

期望输入宽题目或清晰 RQ、稿件、引用表、协议需求
核心动作模式路由、协议模板、证据状态、质量门、可选 offline runtime
关键输出RQ brief、PRISMA 协议骨架、审稿/完整性模板填充、门禁结果

中文示例：「请使用 research-methods，按 systematic_review 协议为主题 [主题] 起草 PRISMA 风格协议骨架。不要编造已完成检索结果或效应量。」

诚实深度：E2/E3 为 parity: partial；E4 管线/实验等为 parity: not_started。契约：skills/research-methods/SKILL.md

4.5 longform-writing — 长文结构与起草

何时使用：需要章节地图、综述结构、证据驱动段落或整篇/多章草稿时。

期望输入文类、读者、主张范围、证据材料、目标结构
核心动作先 section map，再逐节起草并绑定来源与 uncertainty
关键输出结构图、分节草稿、引用/不确定映射、移交编辑清单

中文示例：「请使用 longform-writing。在已有 MERGED_CORE_PAPERS 与证据卡基础上，先输出相关工作章节地图与证据落点，再起草一节；不要新增无来源论断。」

契约：skills/longform-writing/SKILL.md

4.6 academic-editing — 保义学术编辑

何时使用：含义基本正确，需要结构/语言/术语/翻译/审稿回复级修订时。

期望输入待改文本、干预级别、术语表、审稿意见（如有）
核心动作保义润色；不增强 claim strength；记录变更与待确认项
关键输出修订稿、变更说明、术语备注、未决问题

中文示例：「请使用 academic-editing。统一以下段落的中英术语，并压缩空话；保留所有限定词与不确定性，不要新增结果。」

契约：skills/academic-editing/SKILL.md

4.7 prose-lint — 确定性文风检查

何时使用：已有面向人的学术/申请书草稿，需要模板腔、过强表述、空壳强调等规则检查时。

期望输入草稿路径；可选 Vale 命令（WORKFLOW_LAB_PROSE_LINT_CMD）
核心动作按 vale.ini 或 vale-grant.ini 跑规则；结果当审查提示
关键输出发现列表（error/warning/suggestion）；不自动改科学含义

中文示例：「请使用 prose-lint，用申请书配置检查当前立项依据草稿，只列出问题与建议，不要直接改写科学论断。」

配置：src/workflow_lab/config/vale/ · 契约：skills/prose-lint/SKILL.md

EVIDENCE & SAFETY

6. 证据标签与学术安全规则

Optional dual-route discovery remains acceleration-only; see section 3 for primary_only fallback rules.

完整七态词汇见 skills/research-methods/core/contracts/evidence_states.md。公开深研路线至少显式使用 claim / extract / inference / uncertainty；方法包要求七态。

claim extract inference uncertainty missing blocked human_confirmed

允许

  • 带来源定位的摘录
  • 与摘录分离的推断
  • 冲突或弱支持时标 uncertainty
  • 缺材料时标 missing/blocked
  • 提交前要求 human confirmation

禁止

  • 编造论文、DOI、作者、数据、统计量
  • 编造审稿人、实验结果、机构承诺
  • 用摘要/题录冒充全文方法或性能证据
  • 用润色偷偷增强结论强度
  • 把计划写成已完成结果
硬规则：搜索命中 ≠ 已核验证据；解析成功 ≠ 观点正确；lint 通过 ≠ 可投稿。任何 summary 或 abstract 不得当作 full-text evidence。
ESSENTIAL CORE

7. Essential Core 能力与诚实限制

skills/research-methods 打包模式为第一方 essential_core：合约、模式注册表、协议、模板、双 agent 表面与可选离线 runtime。

范围内容parity
E2 方法深度research question / deep research / systematic reviewpartial
E3 论文与诚信academic paper / citation integrity / manuscript reviewpartial
E4 管线与实验academic pipeline / experiment / optional runtime bodiesnot_started
  • parity: partial 表示可运行、有合约与质量门，但不是完整 Academic Research Skills 行为对等。
  • 不是真实多进程编排，不是编辑部权威，不自动安装外部套件。
  • 完整 ARS 为可选外部材料；本仓库不捆绑 ars/ 或 codex/ 树。

可选离线命令示例：

python skills/research-methods/runtime/scripts/essential_full_runtime.py plan --text "..." --json
python skills/research-methods/runtime/scripts/essential_quality_gates.py list --json

P01 · 全局路由

workflow-router
请使用 workflow-router。
我的目标是：[填写目标]。
已有材料：[路径或说明；可写“暂无”]。

P02 · 开题深研

proposal-research · deep research
请使用 proposal-research。
指南/问题：[粘贴或说明]。
研究背景：[填写]。

P03 · 文献发现与候选池

proposal-research · paper discovery
请为主题 [填写] 建立检索策略并执行主路线文献发现。
方向尚不清晰时软目标 80–120 候选；方向清晰时 50–80。

P04 · 可选次路线回退

dual-route fallback
请执行双路线预检并写入 literature_route_status.json。
secondary_provider 可为 null 或其他独立提供方；本环境若不具备次路线条件，请设置 primary_only（或 primary_only_after_secondary_failure），写明 fallback_reason，并继续完成主路线。
不要阻塞等待用户安装后端，也不要编造次路线结果。

P05 · 合并核心文献

MERGED_CORE_PAPERS
请在候选池基础上归一化 DOI/题名/年份、去重、复筛，输出 MERGED_CORE_PAPERS.md 作为收敛后唯一下游权威。
core_papers.md 如需保留，仅作兼容快照并写明权威已切换。

P06 · 优先全文与人工队列

priority full text · OA-first
请从 MERGED_CORE_PAPERS 选择 3–8 篇决策关键文献写入 priority_papers.md。
仅尝试合法 OA-first 获取；每次尝试记入 fulltext_acquisition.csv。
可读全文写 deep_read_report.md；失败项写入 FULLTEXT_MANUAL_ACTION_REQUIRED.md。
禁止描述付费墙绕过；无全文时不得用摘要冒充方法/性能证据。

P07 · 证据抽取

evidence cards
请基于已获材料建立 evidence_cards.jsonl。
每条至少包含 claim / extract / inference 或 uncertainty、来源定位、局限与可用章节。
仅对真实可读文本做 extract；缺全文标 missing。
不要为了凑数量编造卡片或 DOI。

P08 · 系统评价协议规划
