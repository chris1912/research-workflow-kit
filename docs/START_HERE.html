证据边界 / Evidence boundary

本工具包提供可追溯编排，不提供现成科学结论。区分：

claim 可核对后谨慎使用的来源支撑论断 · extract 紧贴原文的摘录 · inference 超出摘录的解释 · uncertainty 冲突、缺全文或弱支持

This kit is a provenance-aware orchestration layer, not a source of finished scientific truth. Keep claim, extract, inference, and uncertainty explicit. Human confirmation is required before final claims, study-design commitments, or submission packaging.

09 HTML 交付门

Entry page + 3–8 decision pages when conclusions are delivered.

10 打包与人工确认

Markdown remains authoritative; Word/PDF optional.

可选双路由与回退 / Optional dual-route and fallback

  • 字段使用提供方中性的 secondary_*；secondary_provider 可为 null、grok 或其他独立提供方。
  • 第二路由仅用于覆盖加速；主路由在第二路由、CLI、鉴权、发现、模型、配额、网络或任务不可用时必须完成。
  • 合并前保持盲法独立发现；随后归一化 DOI/题名/年份、去重、对照遗漏、再筛选，并写入 literature_route_status.json。

Use provider-neutral secondary_* fields. Grok is an example secondary provider, never a required dependency. Missing secondary capabilities produce primary_only (or primary_only_after_secondary_failure) with an explicit reason.

合并核心权威 / Merged-core authority

收敛之后，MERGED_CORE_PAPERS.md 是下游唯一权威。 core_papers.md 仅作为合并前或单路由兼容模板，并须写明权威切换。

After convergence, MERGED_CORE_PAPERS.md is the sole downstream authority. core_papers.md remains a compatible pre-merge or single-route template and must state the authority transition clearly.

优先全文 / Priority full text

从合并核心中选择 3–8 篇决策关键文献，优先合法开放获取；记录每一次尝试；对可读全文做深读；无法获取时建立人工队列。禁止描述付费墙绕过。

Select 3–8 decision-critical papers, attempt lawful OA-first acquisition, log every attempt, deep-read readable full text, and create a manual-action queue when access is unavailable. Never describe paywall bypass.

Templates: skills/proposal-research/templates/priority_papers.md, fulltext_acquisition.csv, FULLTEXT_MANUAL_ACTION_REQUIRED.md, deep_read_report.md.

最终 HTML 门禁 / Final HTML gate

Markdown 与结构化产物仍是权威源。结论型运行还需要：可读 UTF-8 HTML 入口页、3–8 个决策关键页、响应式/可打印导航、可见证据边界文本，以及本地链接检查。

Markdown/structured artifacts remain authoritative. Conclusion-bearing runs also require a readable UTF-8 HTML entry page, 3–8 decision-critical pages, responsive/print-friendly navigation, visible evidence-boundary text, and local-link checks.

Shells and checklist: core_papers_page.html, priority_papers_page.html, html_delivery_checklist.md.

研究方法包何时使用 / When to open research-methods

申报书深研默认走 skills/proposal-research。系统评价协议、引用完整性、同行评审、可重复性规划等走 skills/research-methods 入口。该入口现为第一方 essential_core：合约、模式注册表、模板与可选离线 runtime；E2–E3 协议深度为 parity: partial（研究问题/深研/系统评价 + 论文/完整性/评审），E4 管线/实验/optional_runtime 仍为 parity: not_started；部分深度不等于完整 ARS 行为对等，也不是真实多进程编排。完整 Academic Research Skills 套件为可选外部材料，不随本仓库捆绑，也不是必需运行时依赖。

Use skills/proposal-research for proposal deep research. Route systematic-review, integrity, peer-review, and reproducibility planning through skills/research-methods. That entry is first-party packaging mode essential_core (contracts, mode registry, templates, opt-in offline runtime). E2–E3 protocol bodies are operational at parity: partial (RQ/deep/SR + paper/integrity/review); E4 pipeline/experiment/optional_runtime bodies remain parity: not_started. Partial depth is not full ARS behavioral parity and not real multi-process orchestration. The full Academic Research Skills suite is optional external material: not bundled here and not a required runtime dependency.

Core templates: skills/research-methods/core/templates/prisma_protocol.md, manuscript_review_full.md, citation_integrity_audit.md, reproducibility_checklist.md. Runtime entry: skills/research-methods/runtime/scripts/essential_full_runtime.py. Upstream provenance and license notices remain under docs/THIRD_PARTY_* and docs/licenses/.
