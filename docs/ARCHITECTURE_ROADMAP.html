Research Workflow Kit

这份路线图把分散的科研工具、研究技能、写作模块和质量检查整理成可追溯、可替换、适合公开发布的工作流层，并纳入可选双路由文献发现、合并核心权威、优先全文与结论型 HTML 交付合约。

Grok annotation: Route stages refreshed by Grok on 2026-07-20. Codex annotation: original roadmap 2026-07-15.

旁路发布候选不带虚拟环境可选后端可选第二路由许可证可追溯

工作流主路线

01 研究指南与问题

拆解目标、边界、约束和候选方向。

02 检索策略与预检

关键词、收敛门槛、literature_route_status。

03 文献发现

主路由必做；可选盲法第二路由。

04 合并核心与优先全文

MERGED_CORE_PAPERS 权威；3–8 OA-first。

05 文档解析

把 PDF、DOCX 和图像转成可审计结构化文本。

06 证据组织

证据卡、反证、领域地图；claim/extract/inference/uncertainty。

07 深度研究综合

区分证据、推断、不确定性和研究设计。

08 长文起草与润色

章节结构、段落润色、术语一致。

09 Vale 与 HTML 交付

风格检查；结论型入口页 + 3–8 决策页。

10 打包与人工确认

Markdown 权威；Word/PDF 可选导出。

公共层与可选层

公共层：随仓库上传

  • src/workflow_lab：统一适配器、稳定结果契约和无依赖 CLI。
  • skills：路由、申报书、研究方法、长文、学术编辑和 prose lint 入口。
  • proposal-research 合约模板：双路由状态、合并核心、优先全文、HTML shell。
  • docs：启动手册、路线图、安装、工作流、依赖、来源和许可证边界。
  • Vale 中文规则、格式化器和脱敏最小示例。

可选层：由用户本地安装

  • 文献发现与开放获取检索后端。
  • 独立第二路由提供方（可为 Grok 或其他，非必需）。
  • 高质量文档解析后端。
  • 本地语料库问答后端。
  • 文献计量与领域地图后端。
  • 联网深度研究与模型后端。
  • Vale 可执行文件和模型缓存。

证据层双路由与交付层

Evidence layer

Primary discovery always completes. Secondary is optional coverage only. Blind pools, then normalize DOI/title/year, dedupe, compare omissions, re-screen, and write MERGED_CORE_PAPERS.md.

Delivery layer

Markdown/CSV/JSON remain authoritative. Conclusion-bearing runs add UTF-8 HTML entry navigation, 3–8 decision pages, evidence-boundary text, and local-link checks.

许可证与来源边界

公共目录的改名只改变工作流入口，不抹去第三方版权。原项目名、URL、commit、许可证和修改范围集中记录在 docs/THIRD_PARTY_NOTICES.md 与 docs/THIRD_PARTY_MANIFEST.json。

  • AGPL 运行时默认不复制，只提供适配器和安装说明。
  • CC BY-NC 技能按原非商业条款处理，不能重新标成 MIT。
  • 没有明确许可证的 prompt 参考库只保留外链和用途说明，不复制正文。
  • 真实申请书、个人运行目录、二进制环境、密钥和生成缓存全部排除。
